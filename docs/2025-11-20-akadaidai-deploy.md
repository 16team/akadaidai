# 麺屋 赤橙 ECサイト開発メモ (2025-11-20)

## 1. プロジェクト概要
「麺屋 赤橙」のブランドイメージ（黒×オレンジ）を活かした、モダンでプレミアム感のあるECサイトを構築しました。
最新のWebトレンドである **Bento UI（弁当箱グリッド）** や **Framer Motion** によるアニメーションを取り入れ、単なる通販サイトではなく、ブランドのこだわりを伝える「体験型」のサイトを目指しました。

### 技術スタック
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS v4 (Alpha/Beta features utilized via standard Tailwind classes), CSS Variables for theming
- **Animation**: Framer Motion
- **State Management**: React Context API (Cart)
- **Icons**: Lucide React (via simple SVG implementation)

## 2. 実装済み機能・ページ詳細

### デザイン・UI
- **ブランドカラー**:
  - Main Black: `#0a0a0a` (Rich Black)
  - Brand Orange: `#ea580c` (Vibrant Orange)
- **Bento UI**: 店舗一覧、こだわりページ、商品詳細ページで採用。情報を整理しつつ、視覚的な面白さを提供。
- **レスポンシブ対応**: PC、タブレット、スマートフォン（SP）ですべて崩れずに表示されるよう調整済み。

### ページ構成
1.  **トップページ (`/`)**
    -   **Hero Section**: Framer Motionによる「濃厚、極まる。」のテキストアニメーション。
    -   **Concept**: ブランドコンセプトの提示。
    -   **Product List**: おすすめ商品のグリッド表示。
2.  **商品詳細ページ (`/products/[id]`)**
    -   LP（ランディングページ）風の構成。
    -   Bento UIを用いて「麺」「スープ」「具材」のこだわりを訴求。
    -   カート追加機能（数量選択可能）。
3.  **カート機能 (Drawer)**
    -   画面右側からスライドインするドロワーUI。
    -   `localStorage` を利用したカート内容の保存（ブラウザを閉じても消えない）。
    -   数量変更、削除、小計計算。
4.  **チェックアウト (`/checkout`)**
    -   注文内容の確認。
    -   配送先・支払情報の入力フォーム（UIのみ）。
    -   **模擬決済**: 「注文を確定する」ボタンで完了画面へ遷移（カートクリア処理）。
5.  **店舗一覧 (`/shops`)**
    -   Bento UIによる各店舗（江南本店、一社店、岐阜六条店）のカード表示。
    -   Google Mapsへのリンク。
6.  **こだわり (`/about`)**
    -   ブランドストーリーをBento UIで表現。
    -   スマホ表示時のレイアウト最適化済み。
7.  **法的・情報ページ**
    -   **特定商取引法に基づく表記 (`/legal`)**: 実際の店舗情報を反映済み。
    -   **プライバシーポリシー (`/privacy`)**: 一般的なテンプレートを適用。
    -   **お問い合わせ (`/contact`)**: 入力フォームと送信完了UI（バックエンド未連携）。

## 3. 今後のタスク一覧 (ToDo)

本番運用を開始するために必要な残タスクと、さらに良くするための改善案です。

### 優先度：高（必須）
- [ ] **Stripe決済の本番実装**
    - 現在はモック（見せかけ）です。Stripeアカウントを作成し、APIキーを設定して、実際にクレジットカード決済が通るようにする必要があります。
    - `src/app/api/checkout/route.ts` などのAPIルート作成が必要です。
- [ ] **お問い合わせフォームのバックエンド実装**
    - 現在は送信してもメールが届きません。
    - Resend, SendGrid, または Nodemailer 等を使って、実際に店舗管理者へメールが飛ぶようにする必要があります。
- [ ] **画像の本番差し替え**
    - 現在はAI生成したモック画像（`tsukemen.png`）を使い回しています。
    - 実際の商品のシズル感のある写真（麺、スープ、パッケージ、調理例など）に差し替えることで、コンバージョン率（購入率）が大幅に上がります。
- [ ] **SEO / OGP 設定**
    - `metadata` を各ページで適切に設定する（タイトル、説明文）。
    - SNSでシェアされた時の画像（OGP Image）の設定。
- [ ] **利用規約の作成**
    - 必要に応じて利用規約ページを作成・リンクする。

### 優先度：中（推奨）
- [ ] **CMS (コンテンツ管理システム) の導入**
    - 現在、商品データや店舗情報はコード内（`src/lib/products.ts` 等）に直接書いています。
    - 価格変更や新商品追加をエンジニア以外でも行えるようにするには、**Supabase** や **Contentful**, **MicroCMS** などの導入がおすすめです。
- [ ] **Google Analytics / タグマネージャー導入**
    - アクセス解析のため。
- [ ] **ドメイン設定**
    - Vercel等のホスティングサービスで独自ドメイン（例: `akadaidai.com`）を設定する。

### 優先度：低（将来的な機能拡張）
- [ ] **会員登録 / ログイン機能**
    - 購入履歴の確認、住所入力の省略など。Supabase Auth等が使えます。
- [ ] **在庫管理連携**
    - 実店舗と在庫を共有する場合などの連携。

## 4. 現状の課題・足りていないこと

1.  **バックエンド機能の欠如**:
    - Next.jsはフルスタックフレームワークですが、現在はフロントエンド（見た目と動き）の実装が中心です。データベースやメール送信サーバーとの連携部分は未実装です。
2.  **テスト**:
    - 自動テスト（Unit Test, E2E Test）がありません。長期的な運用の安定性を高めるなら導入を検討すべきです。
3.  **パフォーマンスチューニング**:
    - 画像の最適化はある程度Next.jsがやってくれますが、Lighthouseスコアなどを計測して、さらに高速化する余地があります。

## 5. 開発者への引き継ぎメモ

- **起動方法**: `npm run dev`
- **主要ディレクトリ**:
    - `src/app`: ページルーティング
    - `src/components/ui`: 再利用可能なUIコンポーネント（Button, Bento Grid要素など）
    - `src/lib`: 定数データ（products.ts）、ユーティリティ
- **スタイリング**: Tailwind CSSを使用。`globals.css` でカスタム変数を定義。
- **デプロイ**: Vercel推奨（設定ファイル `next.config.ts` は画像ドメイン設定済み）。
